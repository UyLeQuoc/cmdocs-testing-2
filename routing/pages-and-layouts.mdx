# Pages and Layouts

Next.js uses **file-system routing** — the file structure inside `app/` defines your routes. Special files like `page.tsx`, `layout.tsx`, and `loading.tsx` control how each route behaves.

## Special files

| File | Purpose |
|---|---|
| `page.tsx` | The UI for a route. Makes the route publicly accessible |
| `layout.tsx` | Shared UI that wraps a page and its children |
| `loading.tsx` | Loading UI shown while the page is fetching data |
| `error.tsx` | Error UI shown when the page throws |
| `not-found.tsx` | UI shown for 404 errors |
| `template.tsx` | Like layout, but re-renders on navigation |

<Callout type="info">
  A route is **not** publicly accessible until it has a `page.tsx` file. You can create folders just for organization without creating routes.
</Callout>

## Pages

A `page.tsx` exports a default React component. The folder path becomes the URL:

<Files>
  <Folder name="app" defaultOpen>
    <File name="page.tsx" />
    <Folder name="dashboard" defaultOpen>
      <File name="page.tsx" />
      <Folder name="settings" defaultOpen>
        <File name="page.tsx" />
      </Folder>
    </Folder>
    <Folder name="blog" defaultOpen>
      <File name="page.tsx" />
    </Folder>
  </Folder>
</Files>

| File path | URL |
|---|---|
| `app/page.tsx` | `/` |
| `app/dashboard/page.tsx` | `/dashboard` |
| `app/dashboard/settings/page.tsx` | `/dashboard/settings` |
| `app/blog/page.tsx` | `/blog` |

```tsx title="app/dashboard/page.tsx"
export default function Dashboard() {
  return <h1>Dashboard</h1>
}
```

## Layouts

Layouts wrap pages and preserve state across navigations. The root layout (`app/layout.tsx`) is required and wraps every page in your app.

```tsx title="app/layout.tsx"
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export const metadata = {
  title: 'My App',
  description: 'Built with Next.js',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body className={inter.className}>{children}</body>
    </html>
  )
}
```

### Nested layouts

Add a `layout.tsx` to any route folder to create a nested layout. It wraps all pages in that folder and its children:

<Files>
  <Folder name="app" defaultOpen>
    <File name="layout.tsx" />
    <File name="page.tsx" />
    <Folder name="dashboard" defaultOpen>
      <File name="layout.tsx" />
      <File name="page.tsx" />
      <Folder name="analytics" defaultOpen>
        <File name="page.tsx" />
      </Folder>
    </Folder>
  </Folder>
</Files>

```tsx title="app/dashboard/layout.tsx"
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <div className="dashboard">
      <aside>
        <nav>
          <a href="/dashboard">Overview</a>
          <a href="/dashboard/analytics">Analytics</a>
        </nav>
      </aside>
      <main>{children}</main>
    </div>
  )
}
```

<Mermaid chart="graph TD;
    A[Root Layout] --> B[Dashboard Layout];
    A --> C[Blog Page];
    B --> D[Dashboard Page];
    B --> E[Analytics Page];
    A --> F[Home Page];" />

<Callout type="info">
  Layouts don't re-render when you navigate between sibling pages. This means client-side state (like a search input in a sidebar) is preserved across navigations.
</Callout>

## Route groups

Wrap a folder name in parentheses to create a **route group**. It organizes routes without affecting the URL:

<Files>
  <Folder name="app" defaultOpen>
    <Folder name="(marketing)" defaultOpen>
      <File name="layout.tsx" />
      <Folder name="about" defaultOpen>
        <File name="page.tsx" />
      </Folder>
      <Folder name="pricing" defaultOpen>
        <File name="page.tsx" />
      </Folder>
    </Folder>
    <Folder name="(dashboard)" defaultOpen>
      <File name="layout.tsx" />
      <Folder name="settings" defaultOpen>
        <File name="page.tsx" />
      </Folder>
    </Folder>
  </Folder>
</Files>

| File path | URL |
|---|---|
| `app/(marketing)/about/page.tsx` | `/about` |
| `app/(marketing)/pricing/page.tsx` | `/pricing` |
| `app/(dashboard)/settings/page.tsx` | `/settings` |

The `(marketing)` and `(dashboard)` folders don't appear in the URL but can have their own layouts.

## Loading and error states

Add `loading.tsx` and `error.tsx` for automatic loading and error boundaries:

<Tabs items={["loading.tsx", "error.tsx", "not-found.tsx"]}>
  <Tab value="loading.tsx">
    ```tsx title="app/dashboard/loading.tsx"
    export default function Loading() {
      return (
        <div className="animate-pulse">
          <div className="h-8 bg-gray-200 rounded w-1/3 mb-4" />
          <div className="h-4 bg-gray-200 rounded w-2/3" />
        </div>
      )
    }
    ```

    Next.js wraps your page in a `<Suspense>` boundary using this component. It shows while the page's async data is loading.
  </Tab>
  <Tab value="error.tsx">
    ```tsx title="app/dashboard/error.tsx"
    'use client'

    export default function Error({
      error,
      reset,
    }: {
      error: Error & { digest?: string }
      reset: () => void
    }) {
      return (
        <div>
          <h2>Something went wrong!</h2>
          <button onClick={() => reset()}>Try again</button>
        </div>
      )
    }
    ```

    <Callout type="warn">
      Error components must be Client Components (`'use client'`).
    </Callout>
  </Tab>
  <Tab value="not-found.tsx">
    ```tsx title="app/not-found.tsx"
    import Link from 'next/link'

    export default function NotFound() {
      return (
        <div>
          <h2>Not Found</h2>
          <p>Could not find the requested resource.</p>
          <Link href="/">Return home</Link>
        </div>
      )
    }
    ```
  </Tab>
</Tabs>
