# Data Fetching

Next.js extends the native `fetch` API with caching and revalidation. Combined with Server Components, you can fetch data directly in your components without client-side state management.

## Server vs. Client rendering

<Cards>
  <Card title="Server Components" description="Default. Fetch data on the server, send HTML to the client. Zero client JS." icon="Server" />
  <Card title="Client Components" description="Add 'use client'. Use hooks, effects, and browser APIs. Interactive." icon="Monitor" />
</Cards>

### When to use each

| Feature | Server Component | Client Component |
|---|---|---|
| Fetch data | ✅ `async/await` directly | ❌ Use `useEffect` or a library |
| Access backend resources | ✅ Direct database/API calls | ❌ Must go through an API route |
| Keep secrets on server | ✅ Env vars stay server-side | ❌ Exposed to the client |
| Interactive UI | ❌ No hooks or events | ✅ `useState`, `onClick`, etc. |
| Browser APIs | ❌ No `window`, `document` | ✅ Full access |

## Fetching data on the server

Server Components can be `async`. Use `fetch` or any async operation directly:

```tsx title="app/posts/page.tsx"
export default async function Posts() {
  const res = await fetch('https://api.example.com/posts')
  const posts = await res.json()

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

<Callout type="info">
  You can also use ORMs, database clients, or any Node.js API directly in Server Components. No need for API routes when fetching your own data.
</Callout>

### Multiple fetches

Fetch data in parallel with `Promise.all` to avoid waterfalls:

```tsx title="app/dashboard/page.tsx"
async function getUser() {
  const res = await fetch('https://api.example.com/user')
  return res.json()
}

async function getPosts() {
  const res = await fetch('https://api.example.com/posts')
  return res.json()
}

export default async function Dashboard() {
  // Fetch in parallel — not sequential
  const [user, posts] = await Promise.all([getUser(), getPosts()])

  return (
    <div>
      <h1>Welcome, {user.name}</h1>
      <p>You have {posts.length} posts.</p>
    </div>
  )
}
```

## Fetching data on the client

For interactive features that need client-side data, use the `'use client'` directive:

<Tabs items={["With SWR", "With React Query", "With useEffect"]}>
  <Tab value="With SWR">
    ```tsx title="app/dashboard/stats.tsx"
    'use client'

    import useSWR from 'swr'

    const fetcher = (url: string) => fetch(url).then((r) => r.json())

    export default function Stats() {
      const { data, error, isLoading } = useSWR('/api/stats', fetcher, {
        refreshInterval: 5000,
      })

      if (isLoading) return <div>Loading...</div>
      if (error) return <div>Error loading stats</div>

      return <p>Active users: {data.activeUsers}</p>
    }
    ```

    <Callout type="info">
      SWR handles caching, revalidation, focus tracking, and error retry out of the box.
    </Callout>
  </Tab>
  <Tab value="With React Query">
    ```tsx title="app/dashboard/stats.tsx"
    'use client'

    import { useQuery } from '@tanstack/react-query'

    export default function Stats() {
      const { data, isLoading, error } = useQuery({
        queryKey: ['stats'],
        queryFn: () => fetch('/api/stats').then((r) => r.json()),
        refetchInterval: 5000,
      })

      if (isLoading) return <div>Loading...</div>
      if (error) return <div>Error loading stats</div>

      return <p>Active users: {data.activeUsers}</p>
    }
    ```
  </Tab>
  <Tab value="With useEffect">
    ```tsx title="app/dashboard/stats.tsx"
    'use client'

    import { useState, useEffect } from 'react'

    export default function Stats() {
      const [data, setData] = useState(null)

      useEffect(() => {
        fetch('/api/stats')
          .then((r) => r.json())
          .then(setData)
      }, [])

      if (!data) return <div>Loading...</div>

      return <p>Active users: {data.activeUsers}</p>
    }
    ```

    <Callout type="warn">
      `useEffect` is fine for simple cases, but consider SWR or React Query for production apps — they handle caching, deduplication, and revalidation automatically.
    </Callout>
  </Tab>
</Tabs>

## Composing Server and Client

A common pattern: fetch data in a Server Component and pass it to a Client Component for interactivity.

```tsx title="app/posts/page.tsx (Server)"
import PostList from './post-list'

export default async function Posts() {
  const posts = await fetch('https://api.example.com/posts').then((r) =>
    r.json()
  )

  // Server fetches, client makes it interactive
  return <PostList posts={posts} />
}
```

```tsx title="app/posts/post-list.tsx (Client)"
'use client'

import { useState } from 'react'

export default function PostList({ posts }) {
  const [search, setSearch] = useState('')
  const filtered = posts.filter((p) =>
    p.title.toLowerCase().includes(search.toLowerCase())
  )

  return (
    <div>
      <input
        value={search}
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Search posts..."
      />
      <ul>
        {filtered.map((post) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </div>
  )
}
```
